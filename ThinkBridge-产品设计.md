# ThinkBridge — AI 思维框架教练（产品设计）

> 不是替你思考，而是训练你学会思考。  
> 你把问题扔进来，AI 先判断问题类型，选择合适的思维框架，然后一边示范如何分析，一边解释为什么这样思考，最后给你可复用的自我提问模板。  
> 手机浏览器打开就能用，加桌面就是 APP。

## §1 项目身份

**ThinkBridge**：一个本地自用的移动端 PWA，也是一名 AI 思维框架教练。

用户输入一个真实问题后，DeepSeek 会自动判断问题类型，选择合适的 1-3 个思维框架，带用户完整走一遍思考过程：

1. 这个问题本质上是什么问题
2. 为什么适合用这些框架
3. 每个框架应该按什么顺序思考
4. 每一步为什么要这样问
5. 把用户的问题代入框架示范一遍
6. 最后沉淀成用户以后可以自己使用的提问模板

一句话：**ThinkBridge 不是答案生成器，而是你的私人思维训练教练。**

## §2 本地自用版说明

当前版本只给个人本地使用，不上线、不开放给他人访问。

- 不需要账号系统
- 不需要后端服务
- 不需要数据库
- 不需要云端同步
- 不需要权限系统
- 浏览器直接调用 DeepSeek API（两步 Pipeline）
- API Key 保存在本机浏览器 `localStorage`
- 历史报告保存在 `localForage` / IndexedDB（避免 5MB 限制）

由于本项目暂不公开发布，第一版优先保证：**能用、好用、能帮助自己学习思维框架**。

## §3 设计理念

### 核心洞察一：答案不是最重要的，学会怎么想才重要

普通 AI 工具会直接给建议，但用户下次遇到类似问题，仍然不会自己分析。

ThinkBridge 的目标不是替用户做决定，而是让用户看见：

- 这个问题为什么被归类为决策问题、根因问题、人际问题或规划问题
- 为什么这个框架适合当前问题
- 这个框架先问什么、后问什么
- 每一步背后的思考目的是什么
- 这个框架容易误用在哪里
- 下次不用 AI 时，用户如何自己套用这个框架

### 核心洞察二：多框架不是为了显得复杂，而是为了互相校验

单一框架容易产生盲区。

例如：

- SWOT 能看到优势和风险，但不擅长比较多个选项
- 决策矩阵能比较选项，但容易忽略长期后悔
- 后悔最小化能看长期感受，但可能低估短期风险
- 安全边际能提醒风险缓冲，但可能让人过度保守

真正有价值的部分，是不同框架之间的：

- 共识点
- 矛盾点
- 盲区提示
- 对结论的修正

### 核心洞察三：中文现实语境会改变框架结论

思维框架不能完全照搬西方语境。

分析中国职场、人际关系、创业、家庭、地域选择时，往往还需要考虑：

- 关系网络
- 面子成本
- 时机
- 城市层级
- 监管环境
- 家庭结构
- 性别差异
- 社会评价

这些不是为了制造偏见，而是为了提醒：**现实约束会改变一个建议是否真的可执行。**

## §4 产品目标

### 第一目标：训练用户掌握思维框架

每次输出都要让用户知道：

```text
这个框架解决什么问题？
它的思考顺序是什么？
每一步为什么要这样思考？
我的问题如何代入这个框架？
以后我怎么自己使用它？
```

### 第二目标：给出结构化分析结果

在教学之外，也要给用户一份可直接参考的分析结果，包括：

- 问题理解
- 框架选择
- 多框架分析
- 现实约束检查
- 交叉验证
- 综合建议
- 行动清单

### 第三目标：逐渐减少用户对 AI 的依赖

长期目标不是让用户每次都问 AI，而是让用户逐渐形成自己的思考习惯。

因此每份报告最后都要提供：

- 自我提问模板
- 常见误区
- 迁移练习
- 下次独立思考步骤

## §5 使用模式

### 1. 学习模式（默认）

适合用户想学习思维框架。特点：

- 输出完整教学型报告（11 节标准模板）
- 每个框架都解释底层逻辑和思考顺序
- 每一步都说明为什么这样想
- 多框架交叉验证
- 现实约束检查
- 给出自我提问模板和练习题

### 2. 快速分析模式

适合用户只想尽快得到一个结构化判断。与 AGENTS.md 第 12 节"简短回答模式"是同一模式。

特点：

- 仍走 Step 1 Router 选择框架，但前端不展示完整诊断过程（如框架选择理由、问题类型分析）
- Step 2 使用快速输出模板，内容控制在较短的篇幅内
- 保留 1-2 个核心框架的简要说明
- 重点给判断、风险提醒、行动清单
- 仍然给最小自我提问模板

**快速分析模式输出模板：**

```markdown
## 判断
这个问题属于 ...（一句话）

## 适合框架
用 ... 和 ...，因为 ...

## 分析
（代入用户问题，用框架走一遍核心步骤）

## 风险提醒
- ...
- ...

## 行动建议
1. ...
2. ...
3. ...

## 下次你可以这样想
1. ...
2. ...
3. ...
```

**与学习模式的 Prompt 差异：**

快速分析模式在 Step 2 Analyzer Prompt 中增加：

```text
当前是快速分析模式。请：
- 输出总长度控制在学习模式的 1/3 以内
- 每个框架只保留核心逻辑和 1-2 个教学要点
- 不需要交叉验证章节
- 不需要现实约束检查章节（仅融入分析中）
- 不需要练习题章节
- 必须保留："下次你可以这样想"的自我提问模板
```

## §6 思维框架库（32 个，按领域分类）

完整定义见同目录 `frameworks.md`，按 8 个领域分类：

| 领域 | 数量 | 框架 |
|------|------|------|
| 数字估算 | 3 | 费米推论、二八定律、成本收益分析 |
| 根因分析 | 4 | 第一性原理、五问法、鱼骨图、金字塔原理 |
| 决策 | 4 | SWOT 分析、决策矩阵、OODA 循环、后悔最小化 |
| 系统思维 | 4 | MECE 原则、反馈循环、冰山模型、二阶思维 |
| 创新与突破 | 4 | 逆向思维、类比推理、六顶思考帽、SCAMPER |
| 人际与自我 | 3 | 利益相关者地图、换位思考、成长型思维 |
| 中式思维 | 6 | 中庸思维、关系思维、面子成本、大局观、时机思维、灰度认知 |
| 不可预见性 | 4 | 黑天鹅思维、反脆弱、情景规划、安全边际 |

每个框架包含：解决的问题、思考顺序、常见误用。详见 `frameworks.md`。

## §7 中国社会结构现实约束（不是框架，是过滤器）

以下内容不是为了给人贴标签，而是提醒：同一个建议，在不同现实土壤中可执行性不同。

### 维度一：城乡与城市层级

| 场景 | 需要额外检查的因素 |
|------|------------------|
| 一线城市 | 市场规则、信息效率、竞争强度、机会密度 |
| 新一线/二线 | 市场规则与本地圈层的混合影响 |
| 三四线城市 | 本地关系、资源集中度、信息透明度 |
| 县城及以下 | 熟人网络、声誉成本、本地关键人 |

### 维度二：监管与合规环境

需要检查：

- 是否涉及审批、牌照、资质
- 当地执行是否稳定
- 是否处在监管灰区
- 合规成本是否被低估
- 政策变化会不会打断计划

### 维度三：收入结构与阶层流动

需要检查：

- 收入主要来自工资、资产还是资源
- 所在地区是否存在上升通道
- 行业是否还在增长
- 选择会不会影响未来十年的位置

### 维度四：关系网络与声誉成本

需要检查：

- 谁是关键影响人
- 谁会因为这个选择受损
- 是否需要给对方台阶
- 名声和口碑是否会影响长期收益

### 维度五：身份与性别差异

需要检查：

- 安全成本
- 生育窗口
- 家庭分工
- 社会评价双标
- 职场天花板
- 男性竞争焦虑
- 情感支持网络

## §9 用户流程

```text
打开页面
  → 输入或保存 DeepSeek API Key
  → 选择模式：学习模式 / 快速分析模式
  → 输入问题
  → 点击"开始分析"
  → [Step 1] AI 检查信息完整性，诊断问题类型，选择框架
  → 如果需要补充信息：展示追问，等待用户输入
  → [Step 2] AI 流式输出教学型分析报告
  → 输出自我提问模板和练习题
  → 保存历史记录到 localForage
  → 用户复制报告或回看历史
```

## §11 隐私说明

```text

- API Key 保存在你的本地浏览器。
```

## §12 第一版成功标准

第一版不追求复杂，而是追求这个闭环：

```text
我在手机上打开。
输入一个真实困扰我的问题。
AI 不只给答案，还教我为什么用这个框架。
我能看懂每一步为什么这样思考。
最后我得到一套下次可以自己用的提问模板。
```

如果这个闭环成立，ThinkBridge 就不是普通 AI 问答工具，而是一个真正帮助用户建立思维能力的训练系统。
